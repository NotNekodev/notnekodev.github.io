My Achievements

The Bleed Kernel: Bespoke x86_64 Operating System

I wrote an operating system in C and x86 Assembly that works on real life hardware.
It supports things such as PS/2 Keyboard Input, File Systems, Executing Programs, Memory Management and Allocation, Userspace and has its own Terminal

as you can see in the image below this is the bleed kernel running a standalone terminal program, browsing through the file system and reading files, and even running another program inside the shell, commanded by the user.

SpecSeek: Know everything about your hardware

I wrote an Extreamly Portable and Verbose CPU Specifications Interpreter using a single assembly instruction CPUID as the backend. Many people in the "Hobby Operating Systems" Community wrote working ports for my software to their own execution environments.
Specseek allows for anyone to know exactly what their processor is capable of right down to the individual instructions it supports.

Livefs: the C fileserver

I wrote an extreamly fast static serve file server in C and I was able to achieve less than 1ms server request processing times using performant C code, file caching, Multi-threaded Execution and a Flat Permissions Model

My flat permissions model dictates that permissions are not bound to the individual but it is actaully down to the parent folder, as a result, the higher the folder in the directory heirarchy, the more authority the folder has. this allows for fast setup, cutting out the permissions and authentication time allowing for faster response times. which is fantastic for files or websites that are expected to be accessed by the public or for fast setup over a local network to access shared folders over devices.
